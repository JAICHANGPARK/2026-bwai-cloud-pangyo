# Gemma 4 모델 및 하드웨어 사양 상세 안내

이 문서는 2026년 4월 구글 딥마인드가 발표한 오픈 가중치 경량 모델 패밀리인 **Gemma 4**의 사양과 하드웨어 요구사항을 상세히 다룹니다. 내 하드웨어에 최적인 모델 크기를 파악하고, 로컬 LLM 구동 시의 메모리 동작 원리를 이해하는 데 도움이 됩니다.

---

## Gemma 4 모델 패밀리 소개

Gemma 4는 모바일 디바이스, 개인용 랩톱, 그리고 고성능 워크스테이션에서 가볍고 빠르게 작동할 수 있도록 구글의 최첨단 AI 기술을 집약해 개발된 오픈 가중치 모델 시리즈입니다. 이전 세대에 비해 크게 개선된 아키텍처와 경량화 기법을 적용하여 동급 대비 최상의 추론 속도와 완성도 높은 결과물을 보장합니다.

### 주요 특징
1.  **네이티브 멀티모달(Native Multimodality):** 텍스트뿐만 아니라 **이미지(Vision)와** **오디오(Audio)를** 추가 전처리 없이 직접 입력받아 해석할 수 있습니다.
2.  **구조화된 출력(Structured Output):** JSON 규격을 정확하게 준수하는 응답을 리턴하는 능력이 뛰어나, API 개발 및 로컬 앱 오케스트레이션에 유리합니다.
3.  **향상된 함수 호출(Function Calling):** 로컬 기기 내에서 다양한 외부 API 및 로컬 파일 탐색 함수와 유기적으로 상호작용할 수 있습니다.

## Gemma 4 경량 모델 상세 스펙 (E2B vs E4B)

핸즈온 세션에서 주로 활용하게 될 모델은 온디바이스 및 개인 랩톱에서 원활하게 기동할 수 있도록 최적화된 **Effective (E) 시리즈** 2종입니다.

| 항목 | Gemma 4 E2B (Effective 2B) | Gemma 4 E4B (Effective 4B) |
| :--- | :--- | :--- |
| **실제 매개변수 (Parameters)** | 약 20억 개 | 약 40억 개 |
| **추천 하드웨어** | 시스템 메모리 8GB 이하 노트북 / 인텔 맥 | 시스템 메모리 16GB 이상 노트북 / Apple Silicon |
| **GGUF 파일 크기 (Q4_K_M)** | 약 1.3 ~ 1.5 GB | 약 2.5 ~ 2.8 GB |
| **필요 여유 메모리 (최소)** | 약 3 ~ 4 GB | 약 5 ~ 6 GB (컨텍스트 로드 공간 포함) |
| **주요 활용도** | 실시간 로컬 음성 비서, 가벼운 대화형 API, IoT 단말 | 로컬 복합 코딩 보조, 복잡한 Reasoning, 멀티모달 처리 |
| **Ollama 실행 태그** | `gemma4:e2b` | `gemma4:e4b` |

## 로컬 LLM과 하드웨어 메모리의 관계 (RAM vs VRAM)

로컬에서 AI 모델을 쾌적하게 실행하려면 본인 하드웨어의 메모리 구조를 이해하는 것이 매우 중요합니다.

### 1. Apple Silicon Mac (M 시리즈)
*   **통합 메모리 (Unified Memory):** CPU와 GPU가 물리적으로 하나의 메모리 칩을 공유합니다.
*   **작동 방식:** 모델을 메모리에 한 번만 올려두면 GPU가 복사 오버헤드 없이 바로 계산을 수행하므로, 메모리 대역폭이 극대화되어 엄청나게 빠른 추론 속도를 보입니다.
*   **권장 사항:** 16GB 이상의 통합 메모리를 가졌다면 **E4B** 모델이 아주 부드럽게 기동됩니다. 8GB 램의 기본형 맥북이라면 **E2B**를 강력히 권장합니다.

### 2. Windows PC (GPU 및 CPU 아키텍처)
*   **VRAM (비디오 전용 메모리):** NVIDIA(CUDA) 또는 AMD(ROCm) 그래픽 카드에 탑재된 초고속 메모리입니다.
*   **CPU / RAM (시스템 메모리):** 컴퓨터 본체에 꽂힌 일반 메모리입니다.
*   **작동 방식:** LLM의 연산은 GPU에서 처리할 때 가장 빠릅니다. 따라서 모델 크기가 그래픽 카드의 **VRAM 용량보다 작아야** 최고의 속도가 나옵니다.
    *   *예: 내 그래픽 카드가 RTX 4060 (VRAM 8GB)인 경우, 약 2.8GB 수준의 E4B Q4_K_M 모델은 VRAM에 100% 올라가므로 매우 빠르게 연산됩니다.*
    *   만약 모델 크기가 VRAM 용량을 초과하면, 초과한 만큼 일반 시스템 RAM으로 연산이 넘어가게(Fallback) 되어 텍스트 출력 속도가 급격히 느려집니다.
*   **권장 사항:** 내 외장 GPU VRAM이 4GB 이하라면 **E2B** 모델을, VRAM이 6GB 이상이거나 일반 RAM이 16GB 이상인 PC라면 **E4B** 모델을 권장합니다.

### 3. 인텔 맥북 (Intel Mac)
*   인텔 맥은 내장 그래픽(iris) 또는 라데온 외장 그래픽을 탑재하고 있으나, 최신 AI 연산 가속(Metal/CUDA)의 효율이 Apple Silicon에 비해 크게 떨어집니다.
*   대부분 CPU 단독 연산에 가깝게 동작하므로 발열과 팬 소음이 발생하며 속도가 느릴 수 있습니다. 인텔 맥 사용자분들은 실습 중 쾌적한 반응 속도를 위해 반드시 가벼운 **E2B (`gemma4:e2b`)** 모델을 다운로드해 두시는 것을 권장합니다.
