# DartPad (dartpad.dev) 사용 및 실습 가이드

**DartPad**는 별도의 로컬 개발 환경(Flutter SDK, Android Studio, VS Code 등)을 구성하지 않고도, 웹 브라우저에서 즉시 Dart 문법을 테스트하고 Flutter 앱을 빌드/실행할 수 있도록 구글에서 제공하는 온라인 플레이그라운드입니다.

본 가이드는 Gemma 4 실습의 첫 번째 단계인 **Flutter 웜업(첫 번째 실습)** 과정을 DartPad를 통해 원활하게 진행할 수 있도록 돕기 위해 작성되었습니다.

---

## 1. DartPad 접속 및 기본 환경 구성

1. 웹 브라우저를 열고 [dartpad.dev](https://dartpad.dev)에 접속합니다.
2. 처음 접속하면 기본 Dart 코드가 노출됩니다. Flutter 실습을 위해 좌측 상단의 **[New Pad]** 버튼을 클릭합니다.
3. 팝업창에서 **[Flutter]**를 선택한 뒤 **[Create]** 버튼을 클릭합니다.
   - 이제 화면에 기본적인 Flutter 카운터 예제 코드가 노출되고, 우측에 모바일 화면 모양의 뷰포트(UI 출력 영역)가 준비됩니다.

---

## 2. 실습 코드 실행 단계

1. 생성형 AI 에이전트(Gemini, Cursor, 또는 `agy` CLI)를 활용해 생성된 Flutter 코드 또는 미리 준비된 정답 코드([reference/flutter_app.dart](../hands-on/reference/flutter_app.dart)) 내용 전체를 복사합니다.
2. DartPad 에디터 창(좌측 영역)의 기존 코드를 **전체 선택(Ctrl+A 또는 Cmd+A)한 뒤 붙여넣기(Ctrl+V 또는 Cmd+V)**하여 덮어씁니다.
3. 에디터 상단의 **[Run]** 버튼을 클릭합니다.
4. 잠시 후 우측의 출력 창(UI Output)에 벽돌깨기 게임 등의 플러터 앱이 기동되는 것을 확인할 수 있습니다. 직접 마우스 드래그를 통해 조작해 볼 수 있습니다.

---

## 3. DartPad 핵심 제약사항 및 주의사항

DartPad는 샌드박스 브라우저 환경에서 작동하므로 다음과 같은 제약사항이 있습니다. 코드를 직접 수정하거나 생성할 때 반드시 유의해야 합니다.

### ① 단일 파일 소스코드 (Single-File Output)
- DartPad는 여러 개의 파일 관계를 지원하지 않습니다. 
- 모든 위젯, 상태 관리 클래스, 데이터 모델, 유틸리티 함수 및 `main()` 진입점 함수가 **오직 하나의 코드 창(단일 파일)** 안에 포함되어 있어야 합니다.
- `import 'my_widget.dart';` 와 같은 로컬 상대경로 임포트는 에러를 유발합니다.

### ② 로컬 에셋(Asset) 사용 불가
- 로컬 컴퓨터에 저장된 이미지(`.png`, `.jpg`), 오디오 파일, 혹은 로컬 폰트 파일을 읽어올 수 없습니다.
- **대안 방법:**
  - **이미지:** 웹에 업로드된 URL을 사용해 `Image.network('https://...')` 형태로 로드해야 합니다.
  - **폰트:** `google_fonts` 패키지를 가져와 주입해야 합니다.
  - **아이콘:** 기본 탑재된 `Icons.xxx` 머티리얼 아이콘을 사용하거나 `CustomPainter`를 이용해 직접 도형을 그려야 합니다.

### ③ 사전 허용된 패키지만 사용 가능 (Allowlist)
- DartPad에서는 임의의 외부 패키지(`pub.dev`에 등록된 모든 패키지)를 자유롭게 가져올 수 없고, 구글이 사전 승인한 패키지만 `import` 할 수 있습니다.
- **주요 허용 패키지:**
  - **상태 관리:** `flutter_bloc`, `flutter_riverpod`, `provider`
  - **네트워크:** `httpx`, `dio`, `http`
  - **UI/차트:** `google_fonts`, `fl_chart`, `flutter_svg`, `animations`
  - **게임 엔진:** `flame` (2D 게임용)
  - **기타:** `google_generative_ai` (Gemini API 연동용)

## 4. 문제 해결 (Troubleshooting)

*   **콘솔에 라이브러리 임포트 에러가 납니다:**
    - 허용되지 않은 외부 패키지를 가져왔을 때 발생합니다. 코드 상단의 `import` 문을 확인하고 지원되는 패키지로 변경해 주세요.
*   **Run 버튼을 눌렀는데 화면이 반응이 없거나 하얗게 나옵니다:**
    - 비동기 로직이 루프에 빠졌거나 무한 레이아웃 버그가 일어난 경우일 수 있습니다. 브라우저 페이지를 새로고침(F5)하고 코드를 재검증한 뒤 다시 실행해 보세요.
*   **키보드 포커스가 안 잡힙니다:**
    - DartPad 내부의 모바일 화면(iframe)을 한 번 마우스로 클릭해 포커스를 준 뒤 작동시켜 보세요.
