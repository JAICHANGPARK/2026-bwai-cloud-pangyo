역할: 너는 로컬 LLM 통합에 특화된 시니어 파이썬 개발자야.
작성 목표: 로컬에 기동된 Gemma 4 API 서버(Ollama, LM Studio, llama.cpp) 중 선택한 환경에 맞춰, 도구(Tool/Function) 정보를 모델에 전달하고, 모델이 요청한 도구 호출(Tool Call)을 해석하여 로컬 함수를 실행한 후, 최종 답변을 비동기 스트리밍 방식으로 출력해 주는 파이썬 스크립트를 작성해 줘.

반드시 지켜야 할 제약사항:
1. HTTP 클라이언트 라이브러리로 requests 대신 'httpx'를 사용할 것.
2. 비동기(Async) 코드로 작성하고, 1단계(도구 판단 및 실행)는 비스트리밍(stream=False), 2단계(최종 답변 생성)는 스트리밍(stream=True)으로 동작하도록 구성할 것.
3. 실습용 도구로 `get_current_weather(location)` 함수를 정의하고, 도구 메타데이터 정의(`tools`) 및 바인딩을 포함할 것:
   - 가상의 서울, 도쿄, 뉴욕 날씨 딕셔너리를 기반으로 임의 정보를 리턴하게 구성.
4. 로컬 API 환경 설정을 소스코드 하드코딩이 아닌 '환경변수(os.environ)'를 통해 유연하게 주입할 것:
   - LLM_PROVIDER: 'ollama' | 'lmstudio' | 'llamacpp' (기본값: 'ollama')
   - LLM_API_URL: 해당 서버의 Base URL (기본값은 제공자별 기본 포트 기준 자동 매핑)
   - LLM_MODEL: 호출할 모델 이름 (기본값: 'gemma4:e4b' 또는 다운로드받은 모델 식별자)
5. Ollama의 네이티브 도구 호출 응답 구조와 OpenAI 호환 규격(LM Studio, llama.cpp)의 응답 구조를 각각 분기 처리하여 완벽히 대응할 것:
   - Ollama: 응답 `message` 객체 하위의 `tool_calls` 목록 처리
   - OpenAI 호환: 응답 `choices[0].message` 객체 하위의 `tool_calls` 목록 처리 및 `tool_call_id` 파싱 포함
6. 로컬 서버 연결이 끊겼거나 접속이 안 될 때 상세한 에러 핸들링 코드를 포함할 것.
7. 파일 저장 위치 제약사항:
   - 이 코드는 반드시 이 워크스페이스 내의 'hands-on/function_calling.py' 파일로 작성(또는 덮어쓰기)해야 해. 다른 경로에 생성하면 안 돼.
8. 환경변수 파일 자동 로드:
   - 실습 폴더의 환경변수 설정 파일('.env')을 정상적으로 읽어올 수 있도록, 소스코드 시작 부분에 `python-dotenv` 라이브러리의 `load_dotenv()` 호출을 반드시 포함시켜 줘.
