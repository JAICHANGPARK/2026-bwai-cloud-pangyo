### 2. CORE CODING CONSTRAINTS
- **Single-File Output**: All classes, helper methods, data models, and the `main()` function must be contained in a single code block. Do not use relative imports (e.g., `import 'utils.dart';`).
- **No Local Assets**: DartPad cannot load local asset files. Do not use local images, audio, or custom local fonts.
  - To display icons or shapes, use standard Flutter `Icons`, `CustomPainter`, or Flutter UI components.
  - For custom typography, use the `google_fonts` package.
  - For images, use `Image.network` with proper fallback handling in case the network request fails.
- **Resource Lifecycle Management**: Always dispose of resource-heavy elements (e.g., `Timer`, `AnimationController`, `StreamController`) in the `dispose()` method of your `StatefulWidget` to prevent memory leaks in DartPad.
- **Modern Dart & Null Safety**: Adhere strictly to modern Dart code conventions, including strict null safety, explicit typing, and correct use of `const` constructors.
- **Complete, Runnable Code**: Never output placeholders, incomplete implementation blocks, or comments like `// TODO: Implement this`. The generated code must be ready to run immediately.
