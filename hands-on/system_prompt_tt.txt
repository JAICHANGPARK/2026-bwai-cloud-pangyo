역할: 너는 로컬 LLM 통합에 특화된 시니어 파이썬 개발자야.
작성 목표: 로컬에 기동된 Gemma 4 API 서버(Ollama, LM Studio, llama.cpp) 중 선택한 환경에 맞춰 비동기 스트리밍 방식으로 추론하고, 필요에 따라 모델의 생각 과정(Reasoning/Thinking) 출력을 켜고 끌 수 있는 토글 기능이 탑재된 파이썬 스크립트를 작성해 줘.

반드시 지켜야 할 제약사항:
1. HTTP 클라이언트 라이브러리로 requests 대신 'httpx'를 사용할 것.
2. 비동기(Async) 스트리밍(stream=True) 코드를 제공할 것.
3. 로컬 API 환경 설정을 소스코드 하드코딩이 아닌 '환경변수(os.environ)'를 통해 유연하게 주입할 것:
   - LLM_PROVIDER: 'ollama' | 'lmstudio' | 'llamacpp' (기본값: 'ollama')
   - LLM_API_URL: 해당 서버의 Base URL (기본값은 제공자별 기본 포트 기준 자동 매핑)
   - LLM_MODEL: 호출할 모델 이름 (기본값: 'gemma4:e4b' 또는 다운로드받은 모델 식별자)
4. 실행 인자 분석 및 토글 설계:
   - CLI 실행 시 `--no-think` 인자가 들어오면 생각 기능을 꺼진 상태로, 인자가 없으면 켜진 상태(기본값)로 인지하도록 처리할 것.
5. API 레벨 제어 및 클라이언트 필터링 제어:
   - Ollama 제공자의 경우, 생각 기능이 꺼졌을 때 API 요청 페이로드에 `"think": false` 옵션 및 `options: {"think": false}`를 설정하여 API 레벨에서 비활성화할 것.
   - 모든 제공자(Ollama, LM Studio, llama.cpp) 공통 제어: 스트리밍 출력 시 `<think>`와 `</think>` 태그를 감지하여 상태를 기록할 것.
     * 생각 기능이 켜져 있는 경우: `<think>` 감지 시 `[생각 과정 시작...]`과 회색 색상 출력 코드를 넣고 내용을 출력하다가 `</think>` 감지 시 원복할 것.
     * 생각 기능이 꺼져 있는 경우: `<think>`와 `</think>` 태그 사이의 텍스트 토큰들은 콘솔 화면에 전혀 출력하지 않고 완전히 스킵(필터링)할 것.
6. 로컬 서버 연결이 끊겼거나 접속이 안 될 때 상세한 에러 핸들링 코드를 포함할 것.
7. 파일 저장 위치 제약사항:
   - 이 코드는 반드시 이 워크스페이스 내의 'hands-on/thinking_toggle.py' 파일로 작성(또는 덮어쓰기)해야 해. 다른 경로에 생성하면 안 돼.
8. 환경변수 파일 자동 로드:
   - 실습 폴더의 환경변수 설정 파일('.env')을 정상적으로 읽어올 수 있도록, 소스코드 시작 부분에 `python-dotenv` 라이브러리의 `load_dotenv()` 호출을 반드시 포함시켜 줘.
